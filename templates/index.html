OWASP Top 5 Vulnerability Demo

Explore and learn about the most critical web application security risks

A01: Broken Access Control

Users can act outside their intended permissions

  • • Admin panel accessible without authentication
  • • Direct object reference vulnerabilities
  • • Missing access control checks

A02: Cryptographic Failures

Weak cryptography exposes sensitive data

  • • Hardcoded secret keys
  • • Plaintext password storage
  • • Configuration exposure

A03: Injection

Untrusted data sent to interpreter

  • • SQL injection in login
  • • XSS in user profiles
  • • Command injection

A04: Insecure Design

Missing or ineffective security controls

  • • Poor security architecture
  • • Missing threat modeling
  • • Flawed business logic

A05: Security Misconfiguration

Improper security hardening

  • • Debug mode enabled
  • • Verbose error messages
  • • Default configurations
